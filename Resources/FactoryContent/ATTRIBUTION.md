# Factory content attribution

## NAM models

The bundled `.nam` captures are community captures from the
[pelennor2170/NAM_models](https://github.com/pelennor2170/NAM_models) collection
(GNU GPL v3). See `Models/COPYING-GPL3.txt`.

| Factory file | Original capture name | Capture author (as named in source repo) |
|--------------|----------------------|------------------------------------------|
| `TwinBright_Clean.nam` | Tim R Fender TwinVerb Norm Bright | Tim R |
| `TwinBright_Airy.nam` | Tim R Fender TwinVerb Vibrato Bright | Tim R |
| `Bug333_Clean.nam` | Phillipe P Bug333-Clean-ESR0,002 | Phillipe P |
| `Bug333_CleanCab.nam` | Phillipe P Bug333-Clean-Cab-ESR0,007 | Phillipe P |
| `JCM2000_Clean.nam` | Tim R JCM2000 Clean | Tim R |

These models are provided for out-of-the-box fusion-oriented starting points
(glassy clean to light crunch). You can replace them anytime with any `.nam`
from [TONE3000](https://www.tone3000.com/) or your own captures.

## Cabinet IRs

`Lumen_Glass_212.wav`, `Lumen_Smooth_Lead_412.wav`, and
`Lumen_Tight_Crunch_112.wav` are original LumenDSP synthetic cabinet
impulse responses (MIT, same as LumenDSP application code). They are
designed as transparent fusion-friendly cab colours, not captures of a
specific commercial cabinet.

## Neural Amp Modeler

Real-time inference uses [NeuralAmpModelerCore](https://github.com/sdatkinson/NeuralAmpModelerCore) (MIT).
